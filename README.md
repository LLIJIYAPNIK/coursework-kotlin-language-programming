Консольное приложение на Kotlin для управления базой данных туристических туров, клиентов и бронирований.

## 📋 Описание

Система представляет собой консольное приложение для учёта и управления туристическими турами. Приложение позволяет работать с различными сущностями: турами, клиентами, бронированиями, гидами, транспортом, проживанием и другими компонентами туристического бизнеса.

Данные хранятся в JSON формате, что обеспечивает простоту чтения и редактирования.

## ✨ Возможности

### Основные функции:
- **Управление сущностями** - работа с 20 различными типами данных
- **CRUD операции** - создание, чтение, обновление и удаление записей
- **Поиск** - поиск по любому полю с поддержкой частичного совпадения
- **Сортировка** - сортировка по любому полю (по возрастанию/убыванию)
- **Агрегация** - вычисление sum, avg, min, max для числовых полей
- **Сохранение/загрузка** - работа с JSON файлом для хранения данных
- **Генерация тестовых данных** - автоматическое создание демонстрационных данных

### Поддерживаемые сущности:
- Типы туров (TourType)
- Направления (Destination)
- Туры (Tour)
- Расписания туров (TourSchedule)
- Клиенты (Client)
- Бронирования (Booking)
- Гиды (Guide)
- Назначения гидов (TourGuideAssignment)
- Транспорт (Transport)
- Транспорт туров (TourTransport)
- Проживание (Accommodation)
- Проживание в турах (TourAccommodation)
- Услуги (Service)
- Услуги бронирования (BookingService)
- Платежи (Payment)
- Сотрудники (Employee)
- Сотрудники бронирования (BookingEmployee)
- Отзывы (Review)
- Страховки (Insurance)
- Страховки бронирования (BookingInsurance)

## 🛠 Технологии

- **Kotlin** - основной язык программирования
- **kotlinx.serialization** - сериализация/десериализация JSON
- **JUnit 5** - модульное тестирование
- **Reflection API** - работа с полями классов через рефлексию

## 🚀 Установка и запуск

### Требования:
- JDK 11 или выше
- Gradle или Maven
- IDE (IntelliJ IDEA, VS Code и др.)

### Шаги установки:

1. **Клонируйте репозиторий:**
```bash
git clone https://github.com/LLIJIYAPNIK/coursework-kotlin-language-programming
cd coursework-kotlin-language-programming
```

2. **Соберите проект:**
```bash
./gradlew build
```

3. **Запустите приложение:**
```bash
./gradlew run
```

## 📖 Использование

### Запуск приложения

При запуске приложение автоматически:
1. Проверяет наличие файла `tours_db.json`
2. Если файл отсутствует - генерирует тестовые данные
3. Загружает данные в память
4. Отображает главное меню

### Главное меню

```
СИСТЕМА УЧЁТА ТУРОВ
1. Работа с сущностями
2. Сохранить
0. Выход
```

### Меню сущности

После выбора сущности доступно:
```
Меню сущности: [название_сущности]
1. Показать записи
2. Удалить запись
3. Поиск
4. Сортировка
5. Агрегация
0. Назад
```

### Примеры использования

#### Поиск клиентов:
```
Поле: firstName
Запрос: Иван
```

#### Сортировка туров по цене:
```
Поле сортировки: basePrice
По убыванию? (y/n): y
```

#### Агрегация (подсчёт общей стоимости туров):
```
Поле: basePrice
Функция (sum/avg/min/max): sum
```

## 🔧 Конфигурация

### Изменение пути к файлу БД:
В файле `Main.kt` измените переменную `filepath`:
```kotlin
val filepath = "tours_db.json" // укажите свой путь
```

### Настройки JSON:
В классе `DBManager` можно настроить параметры сериализации:
```kotlin
private val json = Json {
    prettyPrint = true        // красивое форматирование
    ignoreUnknownKeys = true  // игнорировать неизвестные поля
}
```

## 📝 Особенности реализации

### Рефлексия
Приложение использует Kotlin Reflection API для:
- Динамического доступа к полям объектов
- Универсального поиска по любому полю
- Сортировки по любому полю
- Агрегации числовых полей

### Архитектура
Проект реализует принципы:
- **Разделение ответственности** - каждый класс выполняет одну задачу
- **Модульность** - компоненты слабо связаны
- **Тестируемость** - все компоненты покрыты тестами
- **Расширяемость** - легко добавлять новые сущности

## 🐛 Известные ограничения

- Данные хранятся в памяти и загружаются из файла при каждом запуске
- Нет поддержки транзакций
- Поиск регистронезависимый, но не поддерживает регулярные выражения
- Сортировка работает только по строковому представлению значений!!
